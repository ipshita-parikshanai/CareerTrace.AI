# Integration Testing

Given that notification response of the watches/web-hooks is not synchronous and can at least take 1 hour from the time it was created, you can use the `simulation` endpoints to do integration of notification response with your app.

## Simulated API endpoints

### Create a Watch

Endpoint: `/watcher/simulation/watches`

Requirement:

- `notification_endpoint` : an HTTP webserver which can receive POST request
- `event_type_slug` : event for which you want to test the notification
- `event_filters` : filters you want to put in on your event, as defined in [Types of Watches](Crustdata%20Watcher%20API%20126e4a7d95b180259822f83ff2edb672.md)
- `account_filters` : account filters (which are applicable for the event), as defined in [Types of Watches](Crustdata%20Watcher%20API%20126e4a7d95b180259822f83ff2edb672.md)
- `lead_filters` : lead filters (which are valid for the event), as defined in [Types of Watches](Crustdata%20Watcher%20API%20126e4a7d95b180259822f83ff2edb672.md)
- `max_notifications_per_execution` : (optional) maximum number of notifications to send per execution. Must be a multiple of 50 and greater than 0

Why should you use it?

- Sends example notification message to `notification_endpoint` instantly
- Example Request

  ```bash
  curl --location 'https://api.crustdata.com/watcher/simulation/watches' \
  --header 'Content-Type: application/json' \
  --header 'Accept: application/json, text/plain, */*' \
  --header 'Accept-Language: en-US,en;q=0.9' \
  --header 'Authorization: Token $auth_token' \
  --data '{
   "event_type_slug": "job-posting-with-keyword-and-location",
   "event_filters": [
          {"filter_type": "TITLE", "type": "in", "value": "Software Engineer"},
          {"filter_type": "REGION", "type": "in", "value": "Bangladesh"}
      ],
   "account_filters": [
          {"filter_type": "COMPANY_HEADCOUNT", "type": "in", "value": ["10,001+", "5,001-10,000", "11-50", "1-10"]}
      ],
   "notification_endpoint": "http://host.docker.internal:5000/webhook",
   "frequency": 1,
   "expiration_date": "2026-01-01",
   "max_notifications_per_execution": 100
  }'
  ```

# Watching an account

This watch lets you track updates on a specific account in real-time.

## Use Cases

1. Company raised a found: get notified when a specific company raised a funding round
2. Company was mentioned in press: get alerted when a company was mentioned in the news

## Event types

<details id="job-posting-by-company">
<summary>1. Job Posting by Company</summary>

### Job Posting by Company

This event can be used for the following use cases:

1.  **Company is hiring for a job in location**
2.  **Company is hiring for a job with keyword in the description**
3.  **Company posted any job**

- Slug: `company-watch-linkedin-job-postings`
- Event filters:

  | **Filter type**        | **Data type**       | **Description**                                                                                                                                                                                                                                                                                                                                                |
  | ---------------------- | ------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
  | `COMPANY_DOMAIN`       | list(max length: 1) | Domain(s) of the company whose jobs you want to watch. Example: ["artisan.co"] (Optional)                                                                                                                                                                                                                                                                      |
  | `COMPANY_LINKEDIN_URL` | list(max length: 1) | LinkedIn urls of the company whose jobs you want to watch. Example: ["https://linkedin.com/company/signalhq"] (Optional)                                                                                                                                                                                                                                       |
  | `COMPANY_ID`           | list(max length: 1) | Crustdata company IDs of the company whose jobs you want to watch. Example: ["1403180"] (Optional)                                                                                                                                                                                                                                                             |
  | `REGION`               | list(max length: 1) | Specifies the location for the job posting. Example values: ["United States"], ["India"], ["Europe"]. Use [**Filters Autocomplete API**](/docs/2024-11-01/discover/auxiliary-apis/filters-autocomplete) to get valid values.<br/>Full list: [**region_values**](https://crustdata-docs-region-json.s3.us-east-2.amazonaws.com/updated_regions.json) (Optional) |
  | `TITLE`                | list(max length: 1) | Specifies the job title to track. Example values: ["Software Engineer"], ["Product Manager"]. (Optional)                                                                                                                                                                                                                                                       |
  | `DESCRIPTION`          | list(max length: 1) | Specifies keywords that should be present in the job description. Example values: ["Backend Developer"], ["Machine Learning"] (Optional)                                                                                                                                                                                                                       |

- Lead Filters:
  - Not available
- Account Filters: - Not available
</details>

# Watching a lead

This watch lets you track updates on a specific lead in real-time.

## Use Cases

1. Person was promoted to new role: get notified when a specific person is promoted at the same company
2. Person switched jobs and moved to a new company: get alerted when a person changed jobs

## Event Types

<details id="person-promoted-or-changed-jobs">
<summary>1. Person promoted or changed jobs</summary>

### Person promoted or changed jobs

- Slug: `linkedin-person-profile-updates`
- Event filters:

  | **Filter type**              | **Data Type** | **Description**                                                                                                                                                                                                                                |
  | ---------------------------- | ------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
  | `LINKEDIN_PROFILE_URL`       | list          | List of LinkedIn profile urls of the people you want to track. Example: `["https://www.linkedin.com/in/abhilash-chowdhary/", "https://www.linkedin.com/in/manmohitgrewal"]`                                                                    |
  | `FIELDS_TO_TRACK` (optional) | list          | List of fields to track on the people. Possible values are: `['employer_change', 'summary', 'headline', 'skills', 'location']`. Example: `["employer_change", "headline", "skills"]`. If not specified, all fields will be tracked by default. |

- **List of Fields:**
  - `employer_change`: Monitors employment changes (company, title, description, location, dates). Includes new/removed positions.
    - `employer_change.new_positions`: New job positions added
    - `employer_change.removed_positions`: Job positions removed
    - `employer_change.changed_positions.company_name`: Company name changes
    - `employer_change.changed_positions.title`: Job title updates
    - `employer_change.changed_positions.description`: Job description changes
    - `employer_change.changed_positions.location`: Work location updates
    - `employer_change.changed_positions.start_date`: Employment start date changes
    - `employer_change.changed_positions.end_date`: Employment end date changes
  - `summary`: Tracks changes to the profile's "About" section and professional bio
  - `headline`: Monitors updates to the professional tagline below the person's name
  - `skills`: Tracks additions, removals, and changes to listed skills
  - `location`: Monitors changes in geographic location information

- Lead Filters:
  - Not available
- Account Filters: - Not available
</details>
